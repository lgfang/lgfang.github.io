3 What is XML

  • XML stands for EXtensible Markup Language.
  • XML was designed to store and transport data.
  • XML was designed to be both human- and machine-readable.
  • XML can handle structured data (trees)
  • XML has been standardized and loads of tools/libraries are available
    • Editor
    • Parser
    • Validator
    • Other operations

4 XML is a Markup Language

4.1 An XML Example

<?xml version="1.0" encoding="UTF-8"?>
<job xmlns="http://my.com/test/automation/0.1" >
  <web_server addr="http://web.my.com/auto.jsp" />
  <login username="foobar" password="hello"/>
  <test_plan name="example">
    <test_case
        name="atest" author="lgfang"
        feature_id="1234" requirement="NA">
      <action>
        <tl1cmd desc="run a tl1cmd at first"
                tl1_type="new"
                switch_type="MG MGC">
          <cmd_str>rtrv-info-eqpt;</cmd_str>
          <expect>COMPLD</expect>
        </tl1cmd>
     </action>
     <!-- ... -->
    </test_case>
  </test_plan>
</job>

4.2 XML Documents Structure

  • XML declaration (The XML Prolog)
  • A single root element
  • Element
  • Start tag/end tag
    • End tags are merged into start tags if the element has no content.
  • Attribute
  • Content
    • Includes whitespace
    • Special characters (such as "<" etc.) shall be escaped.

4.3 Avoid Element Name Conflicts

  • Use XML namespaces explicitly 
    <root
    xmlns:h="http://www.w3.org/TR/html4/"
    xmlns:f="http://www.w3schools.com/furniture">

  <h:table>
    <h:tr>
      <h:td>Apples</h:td>
      <h:td>Bananas</h:td>
    </h:tr>
  </h:table>

  <f:table>
    <f:name>African Coffee Table</f:name>
    <f:width>80</f:width>
    <f:length>120</f:length>
  </f:table>

</root>
  • Default namespaces 
    <table xmlns="http://www.w3.org/TR/html4/">
  <tr>
    <td>Apples</td>
    <td>Bananas</td>
  </tr>
</table>

5 XML is to define your own language

  • "Well Formed" versus "Valid"
  • Validate according to DTD (Document Type Definition) 
    <?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE note SYSTEM "Note.dtd">
<note>
  <to>Tove</to>
  <from>Jani</from>
  <heading>Reminder</heading>
  <body>Don't forget me this weekend!</body>
</note>
  • Validate according to XSD (XML Schema Definition) 
    <?xml version="1.0" encoding="UTF-8"?>
<note
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:noNamespaceSchemaLocation="address-schema.xsd">
  <to>Tove</to>
  <from>Jani</from>
  <heading>Reminder</heading>
  <body>Don't forget me this weekend!</body>
</note>
  • Other Schema languages for XML
  • XSD vs DTD
    • DTD is obsoleted (?)
    • XSD is newer
    • XSD is more powerful

6 XML Processing

6.1 Processing Models

  • Event-Driven XML Processing
    • Example: SAX
    • pros: speed, efficiency, and memory
    • cons: generally more complex
  • Tree-based XML Processing
    • Example: DOM
    • pros: simple, random access to the entire document (XPath)
    • cons: memory
  • Pull-based XML Processing

6.2 Popular libraries

  • Xerces
  • libxml
  • expat
  • For Python: lxml or built-in xml.* modules
